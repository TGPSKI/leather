# AGENTS-TANNERY.md — leather tannery: curing pipeline, hides, and artifacts

Subagent guide for the tannery domain: event-driven curing workflows, hide
storage, artifact storage, and the curing router.

Load this guide when working on `internal/curing`, `internal/artifact`,
`internal/hide`, or `internal/safepath` (the path-validation helper used by
the hide and artifact stores). For the HTTP handlers that wire these packages
into the `serve` command, see
[AGENTS-SERVE.md](AGENTS-SERVE.md#tannery-mode---tannery). The in-process
`HideBuffer` runtime component lives in the same `internal/hide` package
(`buffer.go`); for its runtime role inside an agent turn, see
[AGENTS-RUNTIME.md](AGENTS-RUNTIME.md). For the scheduler-driven worker
system, see [AGENTS-WORKER.md](AGENTS-WORKER.md).

---

## Vocabulary

| Term | Meaning |
|---|---|
| **Hide** | Raw input bytes received from an external event (webhook payload, direct ingest). Stored on disk in `<hide_dir>/`. |
| **HideKind** | Semantic label on a hide (e.g. `github.pr`, `github.issue`). Used for routing and agent prompt templating. |
| **Cut** | A paged slice of a hide, served to the agent one context-window's worth at a time. |
| **Curing** | A named N-agent pipeline that transforms a hide into a stabilised artifact. |
| **CuringDefinition** | YAML configuration for one curing (agent, queue, page size, retry policy). |
| **Artifact** | Stabilised output of a completed curing run. Stored on disk in `<artifact_dir>/<curing_name>/`. |
| **Router** | First-match mapping from an intake event (source, event type) to a TanneryRoute. |
| **TanneryRoute** | One routing entry: source + optional event_type → hide_kind + curing + queue. |

Full glossary: [docs/GLOSSARY.md](../docs/GLOSSARY.md).

## Package responsibilities

### `internal/hide` — disk storage (`store.go`)

> The in-process `HideBuffer` (`buffer.go`) is documented in
> [AGENTS-RUNTIME.md](AGENTS-RUNTIME.md); this section covers `store.go` only.

`hide.Store` persists incoming event bytes on disk before they are processed.
Each hide is a single file at `<hide_dir>/<id>` with a companion
`<hide_dir>/<id>.meta.json` holding metadata.

Key exported surfaces:

```go
// StoreEntry is the metadata record for one stored hide.
type StoreEntry struct {
    ID        string    // format: "hide_<yyyymmdd>_<HHMM>_<4hex>"
    Kind      string    // semantic label, e.g. "github.pr"
    CreatedAt time.Time
    SizeBytes int64
}

// NewStore returns a Store rooted at dir. The directory is created on first
// write (mode 0700); it need not exist at construction time.
func NewStore(dir string) *Store

// Write stores content as a new hide and returns its metadata entry.
// The hide ID is generated via GenerateHideID().
func (s *Store) Write(kind string, content []byte) (StoreEntry, error)

// Get returns the metadata and raw bytes for the given hide ID.
// Returns os.ErrNotExist if the hide does not exist.
func (s *Store) Get(id string) (StoreEntry, []byte, error)

// List returns metadata for all hides currently in the store.
func (s *Store) List() ([]StoreEntry, error)

// Delete removes the hide and its companion meta file.
// Returns nil if the hide does not exist (idempotent).
func (s *Store) Delete(id string) error

// LoadIntoBuffer reads the hide bytes and wraps them in a HideBuffer.
// pageSize controls how many bytes each Cut returns.
func (s *Store) LoadIntoBuffer(id string, pageSize int) (*HideBuffer, error)
```

#### Storage layout

```
<hide_dir>/
  hide_20260519_1423_af3b           # raw bytes (mode 0600)
  hide_20260519_1423_af3b.meta.json # JSON metadata (mode 0600)
```

#### ID format

`"hide_<yyyymmdd>_<HHMM>_<4hex>"` — generated by `GenerateHideID()`.
The 4-hex suffix provides collision resistance within the same minute.

### `internal/artifact` — artifact storage

`artifact.Store` persists the stabilised output of each curing run.

Key exported surfaces:

```go
// NewStore returns a Store rooted at dir.
func NewStore(dir string) *Store

// Write persists art.Content to disk and updates art.ID and art.CreatedAt.
// Returns the updated Artifact. The curing subdirectory is created lazily.
func (s *Store) Write(art model.Artifact) (model.Artifact, error)

// Get returns the artifact with the given ID (searched across all curing subdirs).
func (s *Store) Get(id string) (model.Artifact, error)

// ListByCuring returns all artifacts under <dir>/<curingName>/ in creation order.
func (s *Store) ListByCuring(curingName string) ([]model.Artifact, error)

// List returns all artifacts across all curing subdirectories.
func (s *Store) List() ([]model.Artifact, error)
```

`model.Artifact` fields:
- `ID string` — format: `"art_<yyyymmdd>_<HHMM>_<4hex>"`
- `CuringName string` — the curing that produced this artifact
- `HideID string` — the hide that was processed
- `Content string` — stabilised text output from the agent
- `CreatedAt int64` — Unix timestamp

#### Storage layout

```
<artifact_dir>/
  <curing_name>/
    art_20260519_1423_bf9c.json   # JSON-encoded model.Artifact (mode 0600)
```

### `internal/curing` — worker, supervisor, router, loader

#### CuringDefinition YAML format

```yaml
name: pr-review
agent: pr-review-agent     # must match a loaded agent definition
queue: default             # named queue to consume
page_size_bytes: 3800      # hide page size passed to HideBuffer.Cut
max_attempts: 3            # retry limit before DLQ routing
timeout_seconds: 120       # per-item processing timeout
```

Fields map to `model.CuringDefinition`.

#### `LoadDir`

```go
// LoadDir reads all *.curing.yaml files from dir.
// Returns valid definitions and a slice of errors for invalid files.
func LoadDir(dir string) ([]model.CuringDefinition, []error)
```

#### Router

`Router` performs first-match routing from an intake event to a `TanneryRoute`.

```go
// NewRouter returns a Router backed by the provided routes (order preserved).
func NewRouter(routes []model.TanneryRoute) *Router

// Match returns the first route whose Source equals source and whose
// EventType is either empty (wildcard) or equals eventType.
func (r *Router) Match(source, eventType string) (model.TanneryRoute, bool)

// Routes returns the ordered route list (copy).
func (r *Router) Routes() []model.TanneryRoute
```

#### CuringWorker — execution pipeline

`Worker` drives one curing: it dequeues items, loads the hide, runs the agent,
writes the artifact, and deletes the hide on success.

```go
// NewWorker creates a Worker for one CuringDefinition.
// concurrency caps simultaneous item processing (semaphore-bounded goroutines).
func NewWorker(def model.CuringDefinition, agents map[string]model.Agent,
    concurrency int, hideStore *hide.Store, artStore *artifact.Store,
    deps *RunnerDeps, qmgr *queue.Manager, router *Router,
    log *logging.Logger) (*Worker, error)

// Run starts the 1-second polling loop. Returns when ctx is cancelled.
func (w *Worker) Run(ctx context.Context)

// ProcessItem processes one QueueItem end-to-end without retry logic.
// Exported for testing and one-shot invocations.
func (w *Worker) ProcessItem(ctx context.Context, item model.QueueItem) error
```

##### `process()` pipeline (unexported)

1. Build a per-call context with `def.TimeoutSeconds` deadline.
2. Lookup the agent in `w.agents`; return `errAgentNotFound` if missing.
3. Call `w.hideStore.LoadIntoBuffer(item.HideID, def.PageSizeBytes)`.
4. Call `buildRunner(buf)` → fresh `runner.Runner` with the hide buffer set.
5. Build `model.TokenBudget{MaxTokens: 32000}` via `buildBudget(def)`.
6. Call `r.Run(ctx, agent, budget)` → agent produces one completion.
7. Call `w.artStore.Write(model.Artifact{...})`.
8. Best-effort: notify configured notifiers; enqueue to output queue.
9. Call `w.hideStore.Delete(item.HideID)`.
10. Return nil.

##### `handleItem()` — retry and DLQ policy

- On `errAgentNotFound`: route immediately to DLQ (no retry).
- On any other error: increment `item.AttemptCount`. If `< def.MaxAttempts`,
  re-enqueue to the same queue. Otherwise route to `<queue>-dlq`.
- DLQ queue name: `<original_queue>-dlq`.

#### Supervisor

`Supervisor` manages one `Worker` per `CuringDefinition`.

```go
// NewSupervisor creates a Supervisor. concMap provides per-queue concurrency
// overrides; definitions with no entry default to concurrency=1.
func NewSupervisor(defs []model.CuringDefinition, agents map[string]model.Agent,
    concMap map[string]model.QueueConcurrencyConfig,
    hideStore *hide.Store, artStore *artifact.Store,
    deps *RunnerDeps, qmgr *queue.Manager, router *Router,
    log *logging.Logger) (*Supervisor, error)

// Start launches one Worker goroutine per definition. Returns immediately.
func (s *Supervisor) Start(ctx context.Context)

// Drain waits for all worker goroutines to exit. Call after ctx cancellation.
func (s *Supervisor) Drain()
```

#### `RunnerDeps` — shared runtime dependencies

```go
type RunnerDeps struct {
    Client        session.LLMClient
    ToolReg       *tool.Registry
    Log           *logging.Logger
    MaxToolRounds int
    Cache         *cache.FileCache
    QueueMgr      *queue.Manager
    Notifiers     map[string]notify.Notifier
    MCPReg        *mcp.Registry
}
```

`RunnerDeps` is constructed once in `initTannery` and shared (by pointer)
across all Workers.

## Operational constraints

### A1 — single-process lock

Only one leather process may own a given `<hide_dir>` at a time. The lock is
acquired via `syscall.Flock(fd, LOCK_EX|LOCK_NB)` on
`<hide_dir>/leather.lock` inside `initTannery`. A second process attempting
to start in tannery mode against the same directory will receive an error
immediately (non-blocking flock) and must exit.

### A2 — backpressure contract

`queue.FileQueue` does not enforce a max depth in the queue itself; depth
limits are advisory and checked by the webhook handler. If `qmgr.Depth(name)`
≥ `tannCfg.Queues[name].MaxDepth`, the webhook handler returns `429 Too Many
Requests` instead of enqueueing.

### At-most-once delivery

Items are dequeued before processing. If the process crashes between dequeue
and artifact write, the item is lost (no write-ahead log). This is an explicit
design trade-off for simplicity; an at-least-once DLQ approach requires
checkpointing and is not implemented in v1.

### Hide deletion timing

Hides are deleted **after** the artifact is successfully written. If the
artifact write fails the item is retried (up to `MaxAttempts`); the hide is
left on disk. After `MaxAttempts` the item is DLQ'd and the hide is *not*
automatically deleted — operators must inspect and clean up manually.

`internal/curing` must never import `internal/cli`.

## Common mistakes

| Mistake | Correct approach |
|---|---|
| Deleting the hide before artifact write | Always write artifact first; delete hide only on `artStore.Write` success |
| Sharing a `runner.Runner` across concurrent items | Build a fresh `runner.Runner` per `process()` call via `buildRunner` |
| Not bounding Worker concurrency | Always use the semaphore channel; never spawn unbounded goroutines |
| Starting two leather processes against the same hide dir | The flock prevents this — ensure `initTannery` error is surfaced as a fatal error |
| Logging hide content or artifact content | Log IDs only; content is user data |
| Using `model.QueueItem.AgentName` for agent resolution | Curing workers use `def.Agent` (the curing-level config), not the queue item agent field |

## Testing patterns

```go
// Standard test helper pattern (package curing, internal):
func testLog(t *testing.T) *logging.Logger { return logging.New("test", model.LogLevelError) }
func testAgent(name string) model.Agent { /* min valid agent */ }
func testDef(name, agentName, queueName string) model.CuringDefinition { /* min valid def */ }

// testWorker constructs a full Worker + stores + queue from t.TempDir().
// Returns (*Worker, *hide.Store, *artifact.Store, *queue.Manager).
func testWorker(t *testing.T, def model.CuringDefinition,
    agents map[string]model.Agent, client session.LLMClient,
    notifiers map[string]notify.Notifier, dir string) (...)

// putHide writes a test hide and returns its StoreEntry.
func putHide(t *testing.T, hs *hide.Store, kind, content string) hide.StoreEntry

// makeItem constructs a QueueItem referencing a hide.
func makeItem(hideID, hideKind, curingName string) model.QueueItem
```

Always use `session.NewMockLLM(session.MockConfig{Response: "...", TokensPerMessage: N})`
as the `LLMClient`. Never call a real model in tests.

For end-to-end pipeline tests spanning the HTTP layer, see
`internal/cli/tannery_e2e_test.go`.

Before opening a PR touching this domain:

- [ ] `go test ./internal/curing/... ./internal/artifact/... ./internal/hide/...` passes
- [ ] `go test -race ./...` is clean
- [ ] Worker tests use `t.TempDir()`; no hardcoded paths
- [ ] `testWorker` helper used for all worker tests (no hand-rolled stores)
- [ ] `MockLLM` used for all LLM calls in tests; no real model calls
- [ ] `TestWorker_ProcessSuccess` pattern covers: artifact written + hide deleted
- [ ] DLQ routing covered by a negative test
- [ ] `errAgentNotFound` triggers DLQ immediately (no retry)
- [ ] `hide.Store.Delete` idempotency tested
- [ ] `artifact.Store.ListByCuring` tested with multiple artifacts
